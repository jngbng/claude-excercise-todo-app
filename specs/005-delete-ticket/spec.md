# 기능 명세: 티켓 삭제

**기능 브랜치**: `005-delete-ticket`

**작성일**: 2026-07-24

**상태**: 초안

**입력**: 사용자 설명: "docs/API_SPECS.md의 DELETE /api/tickets/:id(티켓 삭제, FR-006)를
구현한다. 존재하는 티켓은 영구 삭제하고 204 No Content로 응답하며, 존재하지 않는 ID는 404
NOT_FOUND를 반환한다."

## 사용자 시나리오 및 테스트 *(필수)*

### 사용자 스토리 1 - 불필요한 티켓 제거 (우선순위: P1)

사용자가 더 이상 필요 없는 티켓을 영구적으로 제거할 수 있다.

**우선순위 이유**: 되돌릴 수 없는 파괴적 작업이지만, 다른 조회·수정 기능들이 이미 갖춰진
뒤에도 보드를 깔끔하게 유지하려면 반드시 필요한 기본 기능이다.

**독립 테스트**: 티켓을 삭제한 뒤 같은 티켓을 다시 조회해 정말 사라졌는지 확인하는 것으로
독립적으로 테스트 가능하다.

**인수 시나리오**:

1. **조건**: 존재하는 티켓 / **행동**: 삭제를 요청 / **결과**: 삭제에 성공하고 응답 본문은
   없다.
2. **조건**: 방금 삭제한 티켓 / **행동**: 같은 티켓을 다시 조회 / **결과**: 더 이상 존재하지
   않는 것으로 확인된다.
3. **조건**: 존재하지 않는 티켓 / **행동**: 삭제를 요청 / **결과**: 티켓을 찾을 수 없다는
   사실이 안내된다.

---

### 예외 상황

- 삭제는 복구 기능 없이 영구적이다. 삭제 전 확인 절차(다이얼로그 등)는 이 API의 책임이 아니라
  이를 호출하는 클라이언트의 책임이다.

## 요구사항 *(필수)*

### 기능 요구사항

- **FR-501**: 시스템은 ID로 지정한 티켓을 영구적으로 삭제할 수 있게 해야 한다.
- **FR-502**: 시스템은 삭제된 티켓이 이후 어떤 조회 요청으로도 다시 접근되지 않도록 해야 한다.
- **FR-503**: 시스템은 존재하지 않는 티켓에 대한 삭제 요청에 "찾을 수 없음"으로 응답해야 한다.

### 주요 엔터티

- **Ticket**: `specs/001-create-ticket/spec.md`에서 정의한 것과 동일한 엔터티. 이 기능은
  삭제만 다루며 다른 필드를 변경하지 않는다.

## 성공 기준 *(필수)*

### 측정 가능한 결과

- **SC-501**: 삭제한 티켓은 예외 없이 이후 모든 조회에서 사라진다.
- **SC-502**: 존재하지 않는 티켓 삭제 시도는 항상 명확한 "찾을 수 없음" 안내를 받는다.

## Assumptions

- 인증/권한 체계가 없다는 가정은 `specs/001-create-ticket/spec.md`와 동일하게 유지된다.
- 삭제 확인 UX(다이얼로그 등)는 이 기능의 범위 밖이다.
