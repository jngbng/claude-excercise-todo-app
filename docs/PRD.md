# PRD - Tika (Product Requirements Document)

> 최종 수정: 2026-06-28
> 버전: 1.0 (MVP)

---

## 1. 제품 개요

Tika (Ticket + Kanban)는 개인 할 일 관리를 위한 티켓 기반 칸반 보드 웹 어플리케이션이다.

사용자는 할 일을 카드(티켓) 단위로 생성하고, 칸반 보드에서 드래그앤드롭으로 상태를 이동시키며 업무 흐름을 시각적으로 관리한다. 각 티켓에는 우선순위·날짜 정보를 부여하여 기한 내 업무 완료를 돕는다.

**핵심 가치**:
- **시각적 업무 관리**: 모든 할 일을 칸반 보드 위에서 한눈에 파악한다. 칼럼·카드 배치로 업무 흐름을 시각화하여 현황을 즉시 인지할 수 있다.
- **직관적 조작**: 드래그 앤 드롭 하나로 상태를 변경한다. 복잡한 메뉴 없이 카드를 잡아 옮기는 동작만으로 업무 흐름을 제어한다.
- **개인 생산성**: 우선순위·기한·진행 날짜를 티켓에 직접 기록한다. 오버듀 경고와 완료일 자동 추적으로 일정 관리를 지원한다.

## 2. MVP 범위

### 포함 기능

| 구분 | 내용 |
|------|------|
| 보드 구조 | 고정 4컬럼: **Backlog / TODO / In Progress / Done** |
| 티켓 CRUD | 생성·조회·수정·삭제 |
| 상태 이동 | 드래그 앤 드롭으로 칼럼 간/내 이동 |
| 날짜 관리 | 계획시작일(plannedStartDate), 시작일(startedAt), 계획종료일(dueDate), 종료일(completedAt) |
| 우선순위 | LOW / MEDIUM(기본) / HIGH 뱃지 표시 |
| 오버듀 | 계획종료일 초과 미완료 티켓 시각적 경고 표시 |

### 제외 (2차 스펙)

- 사용자 인증 / 계정 관리
- 커스텀 칼럼 추가·삭제
- 멀티 사용자 / 보드 공유
- 라벨, 코멘트, 파일 업로드
- 알림 / 푸시

## 3. 사용자 시나리오

### 시나리오 1: 할 일 등록

사용자가 "새 티켓" 버튼 클릭 → 제목 입력(필수) → 우선순위/날짜 선택(선택)
→ 저장 → Backlog 칼럼 맨 위에 카드 추가

### 시나리오 2: 할 일 상태 이동

카드 드래그 → TODO 드롭 → 시작일(startedAt) 자동 기록
→ In Progress 드롭 → Done 드롭 → 종료일(completedAt) 자동 기록

### 시나리오 3: 티켓 상세 보기 / 수정
카드 클릭 → 상세 모달 열림
→ 제목/설명/우선순위/날짜 수정 후 저장
→ 또는 삭제 버튼 클릭 → 확인 다이얼로그 → 영구 삭제

### 시나리오 4: 완료 확인
Done 칼럼에서 완료된 카드 목록 확인
→ 24시간 이후 카드는 Done에서 자동으로 표시 제외

### 시나리오 5: 업무 필터링
우측 상단 필터 버튼 2개로 보드에 표시할 티켓 범위를 즉시 좁힌다.

- **이번 주 업무** 버튼 클릭 → 계획시작일(plannedStartDate) 또는 계획종료일(dueDate)이 이번 주(월~일) 안에 해당하는 티켓만 보드에 표시
- **만기일이 지난 업무** 버튼 클릭 → 계획종료일(dueDate)이 오늘 이전이고 완료되지 않은 티켓(isOverdue=true)만 보드에 표시
- 필터 버튼을 다시 클릭하면 필터 해제 → 전체 티켓 복원
- 두 필터는 독립적으로 동작 (동시 적용 불가, 마지막 클릭 기준 단일 선택)

## 4. 핵심 기능 목록

### FR-001: 티켓 생성
- 제목 필수 (최대 200자)
- 설명 (선택, 최대 1000자)
- 우선순위 (Low / Medium / High, 기본값: Medium)
- 시작예정일, 종료예정일 (선택, 한국 날짜 표기, timezone: `Asia/Seoul`)
- 생성 시 기본 상태: Backlog
- 생성 시 칼럼 맨 위 배치 (최신 순)

### FR-002: 보드 조회 (티켓 목록 조회)
- 칸반 보드 형태로 칼럼별 카드 목록 조회: 컬럼 수 4개, position 오름차순 정렬
- 카드에 표시할 정보: 제목, 우선순위 뱃지, 종료예정일, 오버듀 표시
- 각 칼럼 상단에 카드 수 표시

### FR-003: 티켓 상세 조회
- 카드 클릭 시 모달 진입점
- 모달에 제목, 설명, 우선순위, 시작예정일, 종료예정일, 시작일, 종료일, 상태, 생성일 표시

### FR-004: 티켓 수정
- 제목, 설명, 우선순위, 시작예정일, 종료예정일 수정 가능
- 상세 모달 내에서 인라인 편집

### FR-005: 티켓 완료 처리
- Done 칼럼으로 이동 시 completedAt 자동 기록
- Done에서 다른 칼럼으로 복귀 시 completedAt 초기화
- Done 칼럼에는 종료일 기준 24시간 이내 티켓만 표시

### FR-006: 티켓 삭제
- 상세 모달에서 삭제 버튼으로 삭제
- 삭제 전 확인 다이얼로그 표시
- 하드 삭제 (복구 불가)

### FR-007: 드래그 앤 드롭으로 상태/순서 변경
- 칼럼 간 드래그앤드롭으로 상태 변경
- 칼럼 내 드래그앤드롭으로 카드 순서 변경
- 이동 시 서버에 상태 및 순서 동기화
- 낙관적 업데이트 (UI 즉시 반영 → API 성공 확정, 실패 시 롤백)
- TODO 이동 시 startedAt 자동 기록

### FR-008: 오버듀 판정
- 종료예정일(dueDate)이 지났으나 Done이 아닌 카드에 시각적 경고 표시

## 6. 기술 스택

| 기술 | 용도 | 선정 이유 |
|------|------|-----------|
| **Next.js 15** (App Router) | 풀스택 프레임워크 | API Route Handler + React SSR 통합, Vercel 네이티브 지원, 단일 레포로 프론트·백 관리 |
| **TypeScript** (strict mode) | 언어 | 빌드타임 타입 오류 검출, 공유 타입으로 프론트·백 계약 명시화 |
| **React 19** | UI | Concurrent Features, Server Components, 생태계 표준 |
| **Tailwind CSS 4** | 스타일링 | 유틸리티 우선으로 반응형 빠르게 구현, 커스텀 디자인 시스템 없이 일관성 유지 |
| **@dnd-kit** | 드래그 앤 드롭 | 접근성(aria) 기본 지원, 헤드리스 구조로 커스텀 UI 자유도, 터치 이벤트 지원 |
| **Drizzle ORM** | DB 쿼리 | TypeScript-first 스키마 정의, 경량·빠른 번들, 서버리스 환경 최적화 |
| **Vercel Postgres (Neon)** | 데이터베이스 | 서버리스 Postgres, Vercel 네이티브 연동, 콜드스타트 빠름 |
| **Zod** | 요청 검증 | 프론트·백 동일 스키마 공유(`src/shared/`), 런타임 타입 안전성 보장 |
| **Jest + React Testing Library** | 테스트 | React 생태계 표준, TDD 사이클(Red→Green→Refactor) 지원 |
| **Vercel** | 배포 | git push 자동 배포, PR별 프리뷰 환경 자동 생성, HTTPS 기본 제공 |

## 7. 레이아웃 와이어프레임

**구조**: 좌측 Backlog 사이드바 + 우측 3컬럼 보드 Swimlane

```
┌──────────────────────────────────────────────────────────────────┐
│  Tika                                              [+ 새 티켓]   │
├──────────────┬───────────────────────────────────────────────────┤
│  BACKLOG (2) │  TODO (2)     │  IN PROGRESS (1)  │  DONE (1)    │
│  ──────────  │  ──────────   │  ─────────────    │  ─────────   │
│ ┌──────────┐ │ ┌──────────┐  │  ┌─────────────┐  │ ┌─────────┐  │
│ │ 티켓 A   │ │ │ 티켓 C   │  │  │   티켓 E    │  │ │ 티켓 F  │  │
│ │ [MEDIUM] │ │ │ [HIGH]   │  │  │   [LOW]     │  │ │  완료   │  │
│ │ ~12/31   │ │ │ ~12/20   │  │  │             │  │ │   ✓    │  │
│ └──────────┘ │ └──────────┘  │  └─────────────┘  │ └─────────┘  │
│ ┌──────────┐ │ ┌──────────┐  │                   │              │
│ │ 티켓 B   │ │ │ 티켓 D   │  │                   │              │
│ │ ⚠ OVERDUE│ │ │ [MEDIUM] │  │                   │              │
│ └──────────┘ │ └──────────┘  │                   │              │
└──────────────┴───────────────┴───────────────────┴──────────────┘
```

**카드 구성 요소**:
- 제목 (최대 2줄 표시 후 말줄임)
- 우선순위 뱃지 (LOW: 회색, MEDIUM: 파란색, HIGH: 빨간색)
- 계획종료일 (있는 경우)
- 오버듀 경고 아이콘 ⚠ (조건 충족 시)

**모달 (상세/수정)**:
- 카드 클릭 시 오버레이 모달로 상세 정보 표시
- 수정 가능 필드: 제목, 설명, 우선순위, 계획시작일, 계획종료일
- 삭제 버튼 포함 (확인 다이얼로그 → 영구 삭제)
